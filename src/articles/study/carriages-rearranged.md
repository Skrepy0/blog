## 先贴出题目:
### 【问题描述】
列车车厢重排时，位于入轨道和出轨道之间的缓冲轨道按照`FIFO`方式运作， 如下图所示，因此可将它们视为队列。禁止将车厢从缓冲轨道移到入轨道，或从出轨道移到缓冲轨道。所有的车厢移动都要按照图中箭头所示的方向进行。
<Picture

/>

第`k`条轨道`Hk`可直接将车厢从入轨道移到出轨道。其余 `k-1` 条轨道用来缓存不能直接进入出轨道的车厢。

假定有`9`节车厢需要重排，其初始顺序`5、8、1、7、4、2、9、6、3`。假设`k=3`(见图)。3号车厢不能直接进入出轨道，因为1号车厢和2号车厢必须排在3号车厢之前。因此，3号车厢进入缓冲轨道`H1`。6号车厢可进入缓冲轨道`H1`，排在3号车厢之后，因为6号车厢是在3号车厢之后进入出轨道。9号车厢可以继续进入缓冲轨道`H1`，排在6号车厢之后。2号车厢不可排在9号车厢之后，因为它必须在9号车厢之前进入出轨道。因此，2号车厢进入缓冲轨道`H2`.排在第一。4号车厢可以进入缓冲轨道`H2`，排在2号车厢之后。7号车厢也可进入缓冲轨道`H2`，排在4号车厢之后。这时，1号车厢可通过缓冲轨道 `H3`直接进入出轨道。接下来2号车厢从缓冲轨道`H2`进入出轨道，3号厢从缓冲轨道`H1`进入出轨道，4号车厢从缓冲轨道`H2`进入出轨道。由于5号车厢此时仍在入轨道上，且排在8号车厢之后，所以8号车厢进入缓冲轨道`H2`，这样5号车厢可以通过缓冲轨道`H3`，直接从入轨道进入出轨道。然后，6号7号、8号和9号车厢依次从缓冲轨道进入出轨道。

给定入轨道车厢的一个排列，输出至少需要的缓冲轨道数量。

### 【输入形式】
一行整数,之间用一个空格隔开
### 【输出形式】
一行,一个整数,需要的缓冲轨道数量的最小值
### 【样例输入】
```
5 8 1 7 4 2 9 6 3
```
### 【样例输出】
```
3
```

## 问题分析与猜想
每个缓冲轨道上的数都有一个特点,那就是缓冲轨道上的数都是从左向右单调递减的,如在题目的例子中,到1号车厢排的时候缓冲轨道的情况如下:
缓冲轨道`H1`:`9 6 3`
缓冲轨道`H2`:`7 4 2`
而等到1号车厢时,直接从`H3`通过
这里不难发现三条规则:
1. 当某个车厢成为**缓冲轨道的最右边时(即头部)**时，如果它是当前期望输出的最小车厢(例如当前情况下就是1号车厢)，就可以通过缓冲轨道直接送出，否则必须移入某个缓冲队列。
2. 一个车厢能否直接输出，取决于**它左侧是否有比它更小的车厢**。若没有，当它在缓冲轨道的头部时，所有比它小的车厢都已处理完毕，它必然可以直接输出；反之则必须进入缓冲轨道。
3. 进入缓冲轨道的车厢，必须按处理顺序（逆序）放入各缓冲轨道尾部，且保持缓冲轨道内部车厢编号从右向左递增。

我们不妨用另一种过程来解决这个问题:将放入与取出分开来想。
先将全部车厢**放入缓冲轨道**,上面那个例子(`5、8、1、7、4、2、9、6、3`)可以处理成:
缓冲轨道`H1`:`9 6 3`
缓冲轨道`H2`:`8 7 4 2`
缓冲轨道`H3`:`5 1`
然后在从缓冲轨道中取出:
`9 8 7 6 5 4 3 2 1`
由于每个缓冲轨道都是单调递减的,所以我们**每次取缓冲轨道的最右边的车厢就可以做到按顺序取出**,下面可以简单证明一下:
令完成第一步(放入缓冲轨道)后共有k个缓冲轨道,在向外取的过程中,当前将要出缓冲轨道的车厢时`m`号车厢,由于车厢出缓冲轨道的顺序是**按编号先小后大**,因此,`m`号车厢在出缓冲轨道时,前$1\sim m-1$号车厢已经出缓冲轨道;又因为缓冲轨道内部车厢编号从右向左递增,所以当前缓冲轨道中的最小编号车厢必定在某个缓冲轨道的尾部,由数学归纳法可以证明此结论。

因此,我们已经将问题转换成:
**在一个由$1\sim n(n\in N^+,n>1)$不重复数字构成的数组中,划分成若干递减子序列，求最少划分数**

这里**最少划分数**就是这道题的最终答案,我们目前已经将这道题与递减子序列扯上关系了,那么和**最长递增子序列**有什么关系呢?
我们不妨列举一下:
- `1 4 3 2 5`,最长递增子序列长度:3,缓冲轨道最小值:3
- `1 2 3 4 5`,最长递增子序列长度:5,缓冲轨道最小值:5
- `4 3 1 7 2 6 5 8`,最长递增子序列长度:4,缓冲轨道最小值:4
- `5 8 1 7 4 2 9 6 3 10 11`,最长递增子序列长度:5,缓冲轨道最小值:5
>从上面列举可以发现:**最长递增子序列长度与缓冲轨道最小值总是相等的**
因此我们可以这样猜想。

## 进一步推导
想要了解这个猜想的正确性,我们可以先去了解下`Dilworth 定理`
根据这个定理可以推导出:**某各个元素互异的数组的递减子序列的最少划分数等于其最长递增子序列长度**

这也不难理解,这里提供一个提示:**任何一个递增子序列中的两个不同元素不可能同时处于同一个轨道**,

理解了这个后,就不难理解这个结论.

## 结论
最终我们将问题转换成:**在一个由$1\sim n(n\in N^+,n>1)$不重复数字构成的数组中,求其最长递增子序列长度**

## 代码实现
这个问题就变成了一个经典的**动态规划**问题
### 数据输入
首先是数据输入,这个题并没有给车厢总数,因此我们不能用`for`循环,而应该使用`while`循环:
```c++
int x;
while (cin >> x) {
    a.push_back(x);
}
```
### 动态规划部分
#### 定义一维数组dp
首先拆成小的子问题:dp[x]表示以原数组x索引处的数字结尾的**最长递增子序列长度**,
>例如对于数组`a=[1, 4, 3, 2, 5]`,`dp[2]`就表示以数字`3`结尾的最长递增子序列长度,而以数字`3`结尾的最长递增子序列是`[1 3]`,因此`dp[2]=2`

其次,初始化dp数组:显然`dp[0]=1`,第一个数字的最长递增子序列是由它自己构成的长度是1的序列.

#### 状态转移方程
**状态转移方程**可以写为：
$$
dp[i] = \max_{0 \le j < i, \ a[j] < a[i]} (dp[j]) + 1
$$

**方程解读**：
转移逻辑：站在位置`i`，回头找所有在它前面（`j < i`）且数值比它小（`a[j] < a[i]`）的位置`j`，把 `a[i]` 接在 `a[j]` 后面，看哪个接出来的序列最长，在此基础上长度加`1`。

具体代码:
```c++
  for (int i = 1; i < a.size(); i++) {
    int max_ = 0;
    for (int j = i; j >= 0; j--) {
      if (a[i] > a[j]) {
        max_ = max(dp[j], max_);
      }
    }
    dp[i] = max_ + 1;
    res = max(dp[i], res);
  }
```
由于我们要求的是**最长递增子序列长度**,即dp数组中的最大值,因此通过`res = max(dp[i], res)`打擂台求最大值.
最后输出`res`,就得到最终结果了~

### 完整代码
```c++
#include <iostream>
#include <vector>
using namespace std;
int main() {
  vector<int> a;
  int res = 0;
  while (cin >> res) {// 节约内存(但不多qwq)
    a.push_back(res);
  }
  res = 0;
  vector<int> dp(a.size());
  dp[0] = 1;
  for (int i = 1; i < a.size(); i++) {
    int max_ = 0;
    for (int j = i; j >= 0; j--) {
      if (a[i] > a[j]) {
        max_ = max(dp[j], max_);
      }
    }
    dp[i] = max_ + 1;
    res = max(dp[i], res);
  }
  cout << res;
}
```
