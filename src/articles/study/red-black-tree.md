## 二叉搜索树
要了解红黑树,不妨先来了解下**二叉搜索树**(`BST`,`Binary Search Tree`),`BST`是一种有序的二叉树结构，它的核心规则很简单：
**左子树所有节点值 < 根节点 < 右子树所有节点值**。
下面是一个具体的二叉搜索树
```
        8
       / \
      3   10
     / \    \
    1   6    14
       / \   /
      4   7 13
```
---

### 基本性质

1. 每个节点最多有两个子节点
2. 左子树所有值都小于当前节点
3. 右子树所有值都大于当前节点
4. 中序遍历结果是**递增序列**

---

### 常见操作

**1. 查找**

* 从根开始比较
* 小了去左，大了去右
* 平均复杂度：$O(log n)$，最坏 $O(n)$（退化成链表）

**2. 插入**

* 按查找路径找到空位置
* 插入新节点即可

**3. 删除**
分三种情况：

* 叶子节点：直接删除
* 只有一个子节点：用子节点替代
* 有两个子节点：用“中序后继/前驱”替换再删除

:::info 中序后继（Inorder Successor）
中序遍历（左→根→右）时，一个节点后面的那个节点。
对于任意节点，它的后继是：右子树中最左边的节点（即右子树中的最小值）。
:::

:::info 中序前驱（Inorder Predecessor）
中序遍历时，一个节点前面的那个节点。
对于任意节点，它的前驱是：左子树中最右边的节点（即左子树中的最大值）。
:::

---

### 优点

* 结构简单
* 查找、插入、删除平均效率高
* 支持有序遍历

---

### 缺点

* 如果数据有序插入，会退化成链表（性能 $O(n)$）
* 不保证严格平衡

---

### 和红黑树关系

红黑树就是一种“自平衡的 BST”，通过旋转和变色，避免 BST 退化，从而保证最坏情况下仍是 $O(log n)$。

---

下面来说说**红黑树**:
## 定义与概述
### 定义与性质
**红黑树**是一种**自平衡的二叉查找树**（`Binary Search Tree`），每个结点上额外存储一个颜色位（`红色或黑色`）。通过满足一组颜色与结构约束，它能保证树在动态更新后依然保持大致平衡，从而确保查找、插入、删除操作的最坏情况时间复杂度为 $O(log n)$。
> 通俗来讲,红黑树是一种能自动维持平衡的二叉树，它通过给结点涂上**红色或黑色**，并遵守几条简单的“**颜色规则**”，保证树不会歪得太厉害，这样查找、插入、删除数据都很快，就像翻一本有目录且页码分布均匀的书。

那红黑树有什么用途呢?,实际上,红黑树就像一个会自动整理的、有序的大抽屉,找东西快，加/删东西也快.你往里放东西、拿东西、扔掉东西，它都会自己稍微挪一挪，保证你每次找东西永远不费劲。

下面介绍它的6个性质,同时也是红黑树的**数学定义**:
1. 每个结点是红色或黑色(通过名字就能看出来)
2. 根结点是黑色(规定)
3. 所有叶子结点（`NIL`空结点）都是黑色
4. 红色结点的两个子结点必须都是黑色。
（即不允许出现连续的红色结点）
5. 从任一结点到其每个叶子结点的所有路径都包含相同数目的黑色结点。（称为“黑色高度”一致）
6. 红黑树是一颗二叉排序树，左子结点值<根结点值<右(这个就保证了红黑树的有序性,所也查找,增删数据很快,后面会具体讲解)

### 基本术语
- **结点（Node）**:红黑树的基本单元，包含键值、颜色、左子指针、右子指针、父指针。
- **颜色（Color）**:每个结点要么是红色（`Red`），要么是黑色（`Black`）。
- **根结点（Root）**:树的顶层结点，必须是黑色。
- **叶子结点（Leaf / NIL）**:红黑树中所有不存储数据的**空指针位置**，统一视为黑色结点，称为**NIL哨兵**。通常每个实际结点的左右孩子指针都指向 `NIL`（而非 `NULL`）。
- **内部结点（Internal Node）**:存储实际数据的非 `NIL` 结点。

## 红黑树的存储实现

### 代码实现

```cpp
// 红黑树结点定义
struct RBNode {
    int key;                 // 存储的键值
    bool isRed;              // true: 红色, false: 黑色
    RBNode* left;            // 左孩子指针
    RBNode* right;           // 右孩子指针
    RBNode* parent;          // 父结点指针
};

// 红黑树类定义
class RedBlackTree {
private:
    RBNode* root;            // 根结点指针
    RBNode* NIL;             // 哨兵叶子结点（黑色）
};
```

### 1. 颜色表示
- 采用 `bool isRed` 成员，`true` 表示红色，`false` 表示黑色。
- 这种定义直观且节省空间，便于条件判断（如 `if (node->isRed)`）。

### 2. 结点数据
- 示例中使用 `int key` 作为存储数据。实际应用中可定义为 `typedef int Datatype;` 或模板，统一修改即可适配不同数据类型。

### 3. 结点指针
- `left`、`right`、`parent`：三叉链表结构，方便在旋转和修复时快速访问父结点，避免递归或栈。
- 所有指针均指向 `RBNode` 类型，空指针统一使用 `NIL` 哨兵，而非 `nullptr`。

### 4. 哨兵 NIL
- `NIL` 是一个特殊的 `RBNode` 实例，在构造函数中动态创建。
- 其 `isRed = false`（黑色），`left = right = parent = nullptr`（或指向自身，取决于实现）。
- 所有叶子结点的左右孩子均指向 `NIL`，代替传统的 `nullptr`。
- **好处**：无需在旋转、删除等操作中单独判断空指针，统一按黑色结点处理，简化边界条件。

### 5. 根结点 root
- 初始指向 `NIL`，表示空树。
- 插入第一个结点后，`root` 指向该结点，且强制设置 `isRed = false`。

> 通过引入 **黑色哨兵 NIL**，将红黑树的所有外部路径统一为指向同一个黑色结点，从而在代码层面统一处理“空结点”与“真实黑色结点”，使插入、删除、旋转等操作的实现更加简洁、安全。

## 关键操作的具体实现
**红黑树**有四大关键操作:**查找与遍历，旋转与变色，插入与删除，构建红黑树**
### 查找与遍历
基于红黑树的二叉搜索树性质（左子 < 根 < 右子），查找与遍历的思路与普通 `BST`(二叉搜索树) 完全一致，但由于红黑树保持了平衡，操作保证 $O(log n)$ 时间。


#### 查找

1. 从根结点 `root` 开始，当前结点 `cur = root`。
2. 若 `cur == NIL`（哨兵），则查找失败，返回 `false`。
3. 比较目标键 `key` 与 `cur->key`：
   - 若相等则查找成功，返回 `true`（或返回结点指针）。
   - 若 `key < cur->key`则`cur = cur->left`，转步骤 2。
   - 若 `key > cur->key`则`cur = cur->right`，转步骤 2。

> 注意:无需关注结点的颜色，颜色仅用于平衡维护，不影响查找逻辑。

示例代码

```cpp
bool find(int key) {
    RBNode* cur = root;// 第一步
    while (cur != NIL) {// 第二步
        // 第三步
        if (key == cur->key) return true;
        else if (key < cur->key) cur = cur->left;
        else cur = cur->right;
    }
    return false;
}

```


#### 遍历思路（以中序遍历为例）

中序遍历可得到**升序**的键值序列(因为红黑素有左子 < 根 < 右子的性质)。

1. 从根结点开始，递归地：
   - 先遍历左子树（`left`）
   - 访问当前结点（输出 `key` 及颜色）
   - 再遍历右子树（`right`）
2. 遇到 `NIL` 时返回(回溯)。

> 同样忽略颜色，遍历过程与普通 `BST`完全相同。

**示例代码**：

```cpp
void inorderHelper(RBNode* node) {
    if (node == NIL) return;
    inorderHelper(node->left);
    cout << node->key << "(" << (node->isRed ? "R" : "B") << ") ";
    inorderHelper(node->right);
}
```

### 旋转与变色
旋转与变色是红黑树的两大核心调整手段,是红黑树在插入或删除结点后，用于*恢复红黑性质*的两个基本操作。它们都是 **局部操作**，时间复杂度 $O(1)$。

#### 旋转`Rotation`

旋转在不改变二叉搜索树中序遍历顺序的前提下，改变树的局部结构。旋转分为 **左旋** 和 **右旋**，二者互为逆操作。

##### 左旋`Left Rotation`
- **适用场景**：当前结点 `x` 的右子树过高（或出现红色右倾）。
- **操作**：将 `x` 的右子结点 `y` 提升为父结点，`x` 成为 `y` 的左子结点，`y` 原来的左子树变为 `x` 的右子树。
- **效果**：降低 `x` 的深度，增加 `y` 的左子树深度。

```
     x                         y
      \                       / \
       y         -->         x   z
        \
         z
```

##### 右旋`Right Rotation`
- **适用场景**：当前结点 `y` 的左子树过高（或出现红色左倾）。
- **操作**：将 `y` 的左子结点 `x` 提升为父结点，`y` 成为 `x` 的右子结点，`x` 原来的右子树变为 `y` 的左子树。
- **效果**：降低 `y` 的深度，增加 `x` 的右子树深度。

```
       y                     x
      /                     / \
     x         -->         a   y
    / \                         \
   a   b                         b
```

**旋转的本质**：通过调整父子关系和子树挂载位置，改变树的形状，但不破坏二叉搜索树的大小顺序。

#### 变色`Recolor`

- **定义**：仅改变结点的颜色（红 , 黑），不改变树的结构。
- **目的**：在不需要旋转的情况下，通过调整颜色来修复红黑性质的违反（尤其是连续红色或黑色高度不一致）。

**变色的代价**：只修改结点的布尔值，极其轻量。

### 插入与删除
插入和删除是红黑树最复杂的操作，核心思想是：先按普通BST操作，再通过旋转和变色修复被破坏的红黑性质。

#### 插入
插入结点颜色原则：红黑树插入的结点**默认是红色**，因为如果插入的是黑色结点，那么肯定会破坏了黑路同这个特性，调整起来较麻烦，如果插入的是红色结点，**可能不需要调整**，如下图：

**二叉树类型**
首先介绍4种二叉树类型:`LL`、`RR`、`LR`、`RL`

在红黑树的**插入修复**过程中，当新插入的节点 `z` 与父节点 `p`、祖父节点 `g` 形成一条折线或直线时，根据三者的相对位置，可区分为四种类型：**LL、RR、LR、RL**。这些类型决定了修复时采用哪种旋转方向。

> 前提：`z` 的父节点 `p` 为红色（违反性质4），祖父 `g` 为黑色，叔节点 `u` 为黑色（或不存在，即 `NIL` 黑色）。

##### LL 类型（左-左）

**定义**：  
- `p` 是 `g` 的**左子节点**  
- `z` 是 `p` 的**左子节点**  
即：`z`、`p`、`g` 三者形成一条向左倾斜的直线。

**图示**：

```
        g(B)                 p(B)
       /                    /   \
     p(R)      -->         z(R)  g(R)
     /
   z(R)
```

**修复操作**：  
- 对 `g` 进行**右旋**  
- 交换 `p` 与 `g` 的颜色（`p` 变黑，`g` 变红）


##### `RR` 类型（右-右）

**定义**：  
- `p` 是 `g` 的**右子节点**  
- `z` 是 `p` 的**右子节点**  
即：三者形成一条向右倾斜的直线。

**图示**：

```
    g(B)                    p(B)
     \                     /   \
      p(R)      -->       g(R)  z(R)
        \
         z(R)
```

**修复操作**：  
- 对 `g` 进行**左旋**  
- 交换 `p` 与 `g` 的颜色（`p` 变黑，`g` 变红）


##### `LR` 类型（左-右）

**定义**：  
- `p` 是 `g` 的**左子节点**  
- `z` 是 `p` 的**右子节点**  
即：先向左、再向右，形成一个“<”形折线。

```
    g(B)                g(B)                z(R)                z(B)
   /                   /                   /   \               /   \
 p(R)      -->       z(R)      -->       p(R)  g(B)   -->    p(R)  g(R)
   \                 /
    z(R)           p(R)
```

**修复操作**：  
- 先对 `p` 进行**左旋**，得到 LL 类型  
- 然后按 LL 类型处理（对 `g` 右旋 + 变色）


##### `RL` 类型（右-左）

**定义**：  
- `p` 是 `g` 的**右子节点**  
- `z` 是 `p` 的**左子节点**  
即：先向右、再向左，形成一个“>”形折线。

**图示**：

```
    g(B)                g(B)                z(R)                z(B)
     \                   \                 /   \               /   \
      p(R)      -->       z(R)    -->     g(B)  p(R)   -->    g(R)  p(R)
     /                     \
   z(R)                    p(R)
```

#### 删除

```
删除结点
├─ 是叶子结点
│  ├─ 红色 → 直接删
│  └─ 黑色 → 修复（X为双重黑）
│     ├─ S红 → 旋转+变色，转后续
│     ├─ S黑且两子黑 → S变红，上移双重黑
│     ├─ S黑且左子红右子黑 → 右旋S，转下一种
│     └─ S黑且右子红 → 左旋P，变色，结束
├─ 只有左子（必黑，子红） → 子替父，子变黑
├─ 只有右子（必黑，子红） → 子替父，子变黑
└─ 有两子 → 找后继，转上述情况
```


> 以下讨论均假设**被删除结点 D 是父结点 P 的左孩子**。若 D 是右孩子，操作对称（左/右旋方向相反）。  
> 所有空位置均视为黑色 NIL 结点。

---

##### 一、删除的结点是叶子结点（无子结点）

###### 1. 删除结点颜色为红色
- **操作**：直接删除，无需修复。
- **原因**：红色结点不影响黑色高度。
- **示例**：删除下图中红色结点 `20`。

```
        30(B)
       /    \
     20(R)   40(B)
    /   \    /  \
  NIL   NIL NIL NIL
```
删除 `20` 后，树变为：
```
        30(B)
           \
           40(B)
```
此时仍为合法红黑树。

---

##### 2. 删除结点颜色为黑色
删除黑色叶子 `D` 后，其位置被 NIL 代替，该 NIL 视为**双重黑**结点 `X`（`X` 是 `P` 的左孩子）。修复过程根据兄弟 `S` 的情况分为 5 种子情况。

###### 情况 2.1：兄弟结点 `S` 为红色
- **条件**：`S` 红 → `P` 必黑，`SL` 和 `SR` 必黑（NIL）。
- **操作**：
  1. 对 `P` **左旋**
  2. 将 `P` 染红，`S` 染黑
  3. 更新 `S` 为 `P` 的新右孩子（此时 `S` 变为黑色）
  4. 转为后续情况（2.2～2.4）继续处理
- **示例**：删除黑色叶子 `10`，其兄弟 `30` 为红色。

初始树：
```
        20(B)
       /    \
     10(B)   30(R)
    /   \    /  \
  NIL  NIL NIL  NIL
```
删除 `10` 后，`X` 为 `20` 的左 NIL（双重黑）。`S=30`（红）。  
- 左旋 `20`：`30` 成为新根，`20` 变为 `30` 的左子。  
- 变色：`20` 变红，`30` 变黑。  
得到：
```
        30(B)
       /
     20(R)
    /   \
  NIL   NIL
```
此时 `20` 的左孩子仍是双重黑，双重黑已转移到 `20`，需要继续按情况 2.2 处理（`S` 现在为 NIL，视为黑）。标准算法中会继续修复，最终将 `20` 变黑结束。此例仅展示旋转变色过程。

---

###### 情况 2.2：兄弟结点 `S` 为黑色，且 `S` 的两个孩子均为黑色（即无红色孩子）
- **操作**：
  1. 将 `S` 染红
  2. 将双重黑上移到 `P`（即新的 `X = P`）
  3. 若 `P` 原为红色，则将其染黑，结束；否则以 `P` 为新的 `X` 向上继续修复
- **示例**：删除黑色叶子 `10`，兄弟 `30` 为黑且其孩子为黑（NIL）。

初始树：
```
        20(B)
       /    \
     10(B)   30(B)
    /   \    /  \
  NIL  NIL NIL  NIL
```
删除 `10` 后，`X` 为 `20` 的左 NIL（双重黑）。`S=30`（黑），`SL=SR=NIL`（黑）。  
- 将 `S` 变红：`30` 变红。  
- 双重黑上移到 `P`（`20` 变为双重黑）。  
- 由于 `20` 原为黑色，需以 `20` 为新 `X` 继续向上（若 `20` 是根则直接变黑结束）。  
假设 `20` 有父结点，则继续。

---

###### 情况 2.3：兄弟结点 `S` 为黑色，且 `S` 的**左孩子为红色**，右孩子为黑色（近红远黑）
- **操作**：
  1. 对 `S` **右旋**
  2. 交换 `S` 与其左孩子的颜色（左孩子变黑，`S` 变红）
  3. 更新 `S` 为 `P` 的新右孩子（此时 `S` 的右孩子为红色）
  4. 转入情况 2.4
- **示例**：删除 `10`，兄弟 `30` 黑，`SL=25` 红，`SR=NIL` 黑。

初始树：
```
          20(B)
         /    \
       10(B)   30(B)
      /   \    /  \
    NIL  NIL 25(R) NIL
```
删除 `10` 后，`X` 为 `20` 的左 NIL（双重黑）。  
- 对 `S=30` 右旋：`25` 上升，`30` 成为 `25` 的右子。  
- 交换颜色：`25` 变黑，`30` 变红。  
得到：
```
        20(B)
       /    \
     10(B)   25(B)
            /   \
          NIL   30(R)
```
此时新 `S` 为 `25`（黑），其右孩子 `30` 为红，转入情况 2.4。

---

###### 情况 2.4：兄弟结点 `S` 为黑色，且 `S` 的**右孩子为红色**（左孩子任意）
- **操作**：
  1. 对 `P` **左旋**
  2. 将 `P` 的颜色赋给 `S`（`S` 变为 `P` 原来的颜色）
  3. 将 `P` 染黑，并将 `S` 的右孩子染黑
  4. 双重黑消除，结束
- **示例**：承接上例，当前树为：
```
        20(B)
       /    \
     10(B)   25(B)
            /   \
          NIL   30(R)
```
`X` 仍为 `20` 的左 NIL（双重黑），`S=25`（黑），`SR=30`（红）。  
- 对 `P=20` 左旋：`25` 成为新根，`20` 成为 `25` 的左子。  
- `P` 原为黑色，将 `S=25` 染黑（不变），`P=20` 染黑，`SR=30` 染黑。  
得到：
```
        25(B)
       /    \
     20(B)   30(B)
    /   \
   NIL  NIL
```
双重黑消除，树平衡。

---

###### 情况 2.5：兄弟结点 `S` 为黑色，且 `S` 的左右孩子均为红色
- **说明**：此情况实际包含在 2.4 中（因为右孩子为红色即可）。只需按 2.4 操作，左旋 `P`、交换颜色、将右孩子变黑，左孩子虽红但旋转后变为 `P` 的左子，仍为红色，但不会违反性质（因为 `P` 已变黑）。
- **示例**：树结构如下，删除 `10`：
```
          20(B)
         /    \
       10(B)   30(B)
      /   \    /  \
    NIL  NIL 25(R) 35(R)
```
删除 `10` 后，按 2.4 左旋 `20`，`30` 上升，`20` 成为 `30` 左子，交换颜色（`20` 黑变黑？`30` 黑变黑？实际需要将 `P`（20）的颜色赋给 `S`（30），两者原都是黑，所以不变；再将 `20` 染黑，`SR=35` 染黑）。结果：
```
        30(B)
       /    \
     20(B)   35(B)
        \
        25(R)
```
此时树合法。

---

#### 二、删除的结点只有左子结点，没有右子结点

- **前提**：此类结点必为黑色（若为红色，其黑色子结点会破坏性质5），其左子结点必为红色。
- **操作**：
  1. 用左子结点替换待删结点
  2. 将左子结点染黑
  3. 删除完成
- **示例**：删除黑色结点 `20`，它只有红色左子 `15`。

```
        30(B)
       /    \
     20(B)   40(B)
    /   \
  15(R)  NIL
```
删除 `20`，用 `15` 替换并染黑：
```
        30(B)
       /    \
     15(B)   40(B)
```
此时树合法。

---

#### 三、删除的结点只有右子结点，没有左子结点

- **前提**：对称情况，待删结点为黑色，右子结点为红色。
- **操作**：
  1. 用右子结点替换待删结点
  2. 将右子结点染黑
  3. 删除完成
- **示例**：删除黑色结点 `20`，它只有红色右子 `25`。

```
        30(B)
       /    \
     20(B)   40(B)
    /   \
  NIL   25(R)
```
删除 `20`，用 `25` 替换并染黑：
```
        30(B)
       /    \
     25(B)   40(B)
```

---

#### 四、删除的结点既有左子结点，又有右子结点

- **策略**：转化为删除**后继结点**（中序后继，即右子树中最小的结点）或前驱。
- **步骤**：
  1. 找到待删结点 `D` 的中序后继 `Y`（`Y` 最多只有一个右子结点，且 `Y` 的颜色任意）
  2. 将 `Y` 的键值复制到 `D`（不改变颜色）
  3. 转而删除 `Y`（此时 `Y` 属于情况一、二或三）
  4. 按对应情况修复
- **示例**：删除既有左子又有右子的黑色结点 `20`。

初始树：
```
        20(B)
       /    \
     10(R)   30(R)
    /   \    /  \
  NIL  NIL 25(B) 40(B)
```
中序后继为 `25`（黑色，叶子）。  
- 将 `25` 的值复制到 `20`，树变为：
```
        25(B)
       /    \
     10(R)   30(R)
    /   \    /  \
  NIL  NIL 25(B) 40(B)
```
- 删除原来的 `25` 结点（黑色叶子）。按情况 1.2 修复（兄弟 `40` 黑且无红孩子，具体过程略）。最终得到合法红黑树。
### 构建红黑树
构建一棵红黑树，通常采用**动态插入**的方式：从空树开始，逐个插入节点，并在每次插入后通过旋转和变色修复红黑性质。
#### 初始化
创建一个空的红黑树，包含一个黑色哨兵节点 `NIL`（所有叶子指向它）。

根节点 `root` 初始指向 `NIL`。

#### 插入节点（核心构建过程）
对于每个待插入的键值 `key`，执行以下步骤：

1. 标准`BST`插入
按**二叉搜索树**规则找到插入位置，将新节点 `z` 插入为红色节点。

设置 `z` 的左右孩子为 `NIL`，颜色为红色。

2. 修复红黑性质（`insertFixup`）
如果 `z` 的父节点 `p` 是黑色，无需修复。
如果 `p` 是红色（违反性质4），则根据叔叔节点 `u` 的颜色分情况处理：

| 情况   | 叔叔 `u` 颜色                 | 操作                                                                 | 后续             |
|--------|------------------------------|----------------------------------------------------------------------|------------------|
| Case 1 | 红色                         | ① 父、叔变黑，祖父变红；<br>② `z` 指向祖父，继续检查                  | 可能向上传播     |
| Case 2 | 黑色，且 `z` 是父的**内侧**孩子 | ① 对父节点单旋（使 `z` 外翻）；<br>② 转为 Case 3                       | 继续             |
| Case 3 | 黑色，且 `z` 是父的**外侧**孩子 | ① 对祖父反向旋转；<br>② 交换父与祖父颜色                               | 结束             |

3. 根节点强制黑色
修复结束后，将 `root` 颜色设为黑色。

#### 构建红黑树的具体例子
举例: 从空树开始，依次插入 **10, 5, 20**，逐步调整。

1. **插入 10**
- 空树，插入为根，红色 → 违反性质2，染黑。
```
10(B)
```

2. **插入 5**
- 5 < 10，作为左孩子，红色。父为黑 → 无需调整。
```
  10(B)
 /
5(R)
```

3. **插入 20**
- 20 > 10，作为右孩子，红色。父为黑 → 无需调整。
```
    10(B)
   /    \
 5(R)   20(R)
```

此时树已满足所有红黑性质，无需进一步调整。

---

**最终树**：根10黑色，左右孩子均为红色，合法。

---

**如果再插入 15（演示修复）**

继续插入 **15**：
- 15 > 10，进入右子树；15 < 20，作为20的左孩子，红色。
- 父20为红色，叔叔5为红色 → **Case 1**（叔红）：
  - 父(20)变黑，叔(5)变黑，祖父(10)变红。
  - 祖父10变为红色，且它是根 → 强制染黑。
```
      10(B)
     /    \
   5(B)   20(B)
         /
       15(R)
```

完成。
## 红黑树的应用

### 核心优势
- 插入、删除、查找都是 $O(log n)$，且中序遍历得到有序序列。
- 相比 `AVL` 树，插入/删除旋转次数更少；相比跳表，内存更紧凑。

### 典型应用场景

| 领域 | 具体例子 | 说明 |
|------|----------|------|
| 标准库 | C++ `std::map`、Java `TreeMap` | 有序键值对，动态增删查 |
| 操作系统 | Linux CFS 调度器、内存区域管理 | 按 vruntime 排序进程，快速查找相邻区间 |
| 数据库 | Redis 有序集合（zset，数据量大时用跳表，但原理类似） | 支持范围查询、排名操作 |
| 定时器 | 心跳超时管理 | 按超时时间排序，快速取出最早到期事件 |

### 示例问题：订单价格区间查询

**问题**：电商订单频繁插入/删除/修改，要求快速查出价格在 $[L,R]$ 内的所有订单。

**基于红黑树的解法**：
1. 以订单**价格**为 `key` 建立红黑树，每个节点存储该价格下的订单列表（处理价格重复）。
2. **插入/删除**：$O(log n)$。
3. **区间查询**：
   - 从根开始找到第一个价格 ≥ L 的节点（$O(log n)$）。
   - 中序遍历，收集价格 ≤ R 的订单，直到超过 `R`。遍历 `k` 个结果需要 $O(k)$ 时间。
   - 总复杂度 $O(log n + k)$，比全表扫描快得多。
